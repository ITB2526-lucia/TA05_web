Entorno Virtualizado con Proxmox y Active Directory

Este proyecto consiste en la creación de un entorno virtualizado completo utilizando Proxmox como hipervisor. El objetivo fue diseñar una infraestructura flexible, escalable y segura que permitiera gestionar máquinas virtuales, implementar un dominio Active Directory y simular un entorno empresarial real.

Objetivos del proyecto

El proyecto se centró en construir un entorno profesional que permitiera:

  • Gestionar máquinas virtuales desde un único panel centralizado.
  • Implementar un dominio Active Directory funcional.
  • Crear políticas de grupo (GPO) para usuarios y equipos.
  • Configurar redes internas segmentadas para mejorar la seguridad.
  • Simular un entorno empresarial real con clientes Windows y servidores Linux.

Arquitectura del sistema

El sistema se basa en un servidor Proxmox que aloja varias máquinas virtuales. Entre ellas se encuentran:

  • Un controlador de dominio Windows Server.
  • Clientes Windows 10 unidos al dominio.
  • Servidores Linux para pruebas y servicios adicionales.
  • Redes internas separadas para gestión, usuarios y servicios.

Esta arquitectura permite aislar servicios, mejorar la seguridad y facilitar la administración del entorno.

Active Directory y políticas de grupo

Se configuró un dominio completo con usuarios, grupos y políticas de seguridad. Las GPO permiten controlar configuraciones clave dentro de la red, como:

  • Restricciones de acceso y permisos.
  • Instalación automática de software.
  • Mapeo de unidades de red.
  • Configuración del escritorio y directivas de seguridad.

Gracias a estas políticas, el entorno se comporta como una red empresarial real, con control centralizado y gestión eficiente de usuarios y equipos.

Tecnologías utilizadas

Para llevar a cabo este proyecto se utilizaron diversas herramientas y tecnologías que permiten construir un entorno virtualizado profesional y escalable. Entre ellas destacan:

  • Proxmox VE como hipervisor principal.
  • Windows Server para la implementación de Active Directory.
  • Windows 10 como clientes unidos al dominio.
  • Distribuciones Linux para servicios adicionales y pruebas.
  • Redes virtuales internas y bridges configurados en Proxmox.
  • GPO para la gestión centralizada de usuarios y equipos.

Conclusiones

Este proyecto me permitió comprender en profundidad cómo funciona un entorno empresarial real, desde la gestión de máquinas virtuales hasta la administración de usuarios mediante Active Directory. También reforzó mis conocimientos en redes, seguridad y virtualización, y me dio experiencia práctica en la creación de infraestructuras escalables y bien organizadas.

Gracias a este entorno pude simular situaciones reales, aplicar políticas de seguridad, automatizar configuraciones y trabajar con diferentes sistemas operativos de forma integrada. Es un proyecto que refleja mi capacidad para diseñar, implementar y administrar sistemas complejos.
