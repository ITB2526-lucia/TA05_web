Virtualized Environment with Proxmox and Active Directory

This project consists of creating a complete virtualized environment using Proxmox as the hypervisor. The goal was to design a flexible, scalable, and secure infrastructure that would allow the management of virtual machines, the implementation of an Active Directory domain, and the simulation of a real business environment.

Project Objectives

The project focused on building a professional environment that would allow:

  • Managing virtual machines from a single centralized panel.
  • Implementing a functional Active Directory domain.
  • Creating Group Policies (GPO) for users and computers.
  • Configuring segmented internal networks to improve security.
  • Simulating a real business environment with Windows clients and Linux servers.

System Architecture

The system is based on a Proxmox server hosting several virtual machines. Among them:

  • A Windows Server domain controller.
  • Windows 10 clients joined to the domain.
  • Linux servers for testing and additional services.
  • Separate internal networks for management, users, and services.

This architecture allows service isolation, improved security, and easier environment administration.

Active Directory and Group Policies

A complete domain was configured with users, groups, and security policies. GPOs allow control over key configurations within the network, such as:

  • Access restrictions and permissions.
  • Automatic software installation.
  • Network drive mapping.
  • Desktop configuration and security directives.

Thanks to these policies, the environment behaves like a real business network, with centralized control and efficient management of users and devices.

Technologies Used

To carry out this project, various tools and technologies were used to build a professional and scalable virtualized environment. Among them:

  • Proxmox VE as the main hypervisor.
  • Windows Server for Active Directory implementation.
  • Windows 10 as domain-joined clients.
  • Linux distributions for additional services and testing.
  • Internal virtual networks and bridges configured in Proxmox.
  • GPO for centralized management of users and devices.

Conclusions

This project allowed me to deeply understand how a real business environment works, from virtual machine management to user administration through Active Directory. It also strengthened my knowledge in networking, security, and virtualization, and gave me practical experience in creating scalable and well-organized infrastructures.

Thanks to this environment, I was able to simulate real situations, apply security policies, automate configurations, and work with different operating systems in an integrated way. It is a project that reflects my ability to design, implement, and manage complex systems.
